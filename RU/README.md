# CCS Hub Optimization Tool (v1.0)

## Назначение инструмента (Overview)
Данный проект представляет собой математическую модель (Two-Stage MILP), разработанную для оптимизации графика подключения промышленных источников CO₂ к центральному хабу улавливания и хранения (CCS Hub). 
Система максимизирует общий объем захваченного CO₂ за заданный горизонт планирования, а затем минимизирует совокупные затраты на захват, строго соблюдая ограничения на пропускную способность трубопровода (Annual Hub Capacity), ёмкость хранилища (Cumulative Storage Capacity) и минимальный срок жизни подключений (Minimum Connection Time).

## Поддерживаемые форматы данных (I/O Formats)

### Входные форматы (Импорт сценариев)
**Примечание о параметре `Remaining Life`:** В модели остаточный срок жизни трактуется строго от **начала горизонта планирования** (например, с 2025 года), а не от момента подключения к трубе. Если горизонт 25 лет, а оставшийся срок жизни установки `L = 15`, то при подключении её на 5-м году установки она сможет проработать с CCS только $15 - 5 = 10$ лет.

- **JSON (`.json`)**: Классический формат для полного сохранения состояния сессии. Включает в себя системные параметры и список установок со всеми предрассчитанными лимитами.
- **Excel (`.xlsx`)**: Основной и самый удобный пользовательский формат. Файл должен состоять из двух листов:
  - `SystemParameters` (Колонки: `Parameter`, `Value`)
  - `Plants` (Колонки: `id`, `name`, `co2_flow_mtpy`, `capture_cost_euro_per_t`, `remaining_life_y`)
- **CSV (`.csv`)**: Поддерживается только для импорта списка установок (аналог листа `Plants`). Удобно для массовой загрузки данных об источниках выбросов. Системные параметры при этом не перезаписываются и могут настраиваться в UI отдельно.

### Выходные форматы (Экспорт результатов)
Независимо от формата, результаты содержат детализированную аналитику, объединяющую общие агрегаты, годовую динамику хаба и объяснения решений по каждой установке (Rule-Based Explainer).
- **JSON (`.json`)**: Полный машиночитаемый экспорт всех таблиц и структур единым файлом.
- **Excel (`.xlsx`)**: Экспорт в один Excel-файл с 5 вкладками:
  1. `Summary` — системные метрики (итоговая цена, объемы, остаток хранилища).
  2. `PlantResults` — решение по каждой установке (выбрана ли, стоимость, год старта).
  3. `YearlyResults` — динамика хаба по годам (загрузка, свободное место).
  4. `PlantYearContributions` — матрица активности (какая установка дает вклад в конкретный год).
  5. `DecisionExplanations` — текстовое интеллектуальное объяснение, почему установка была выбрана именно в этот год (или почему не попала в решение).
- **CSV ZIP Archive (`.zip`)**: Загрузка списка сырых `.csv` файлов (по одному на каждую таблицу из списка выше), запакованных в единый ZIP-архив.

## Запуск интерфейса (Web UI)
Проект снабжен современным графическим веб-интерфейсом на базе Streamlit.
1. Откройте терминал в папке проекта.
2. Активируйте виртуальное окружение: `.\venv\Scripts\activate`
3. Запустите приложение:
   ```bash
   streamlit run app.py
   ```
4. В браузере (обычно `http://localhost:8501`) откроется инструмент.
5. Для быстрого старта загрузите `example_scenario.xlsx` или скачайте шаблон ("Download Excel Template") в боковой панели. Расчет происходит по нажатию кнопки во вкладке "Optimization Summary".

---

## Запуск в консоли (CLI)
Если вы хотите запустить расчет без графического интерфейса (headless-режим):
```bash
python main.py example_scenario.xlsx
```
*Поддерживаются форматы `.json`, `.xlsx`. Скрипт выведет процесс решения в консоль и сохранит результат в текущую папку под именем `results.json`.*

## Требования к окружению (Dependencies)
Проект работает на Python 3.9+ и использует фреймворки, зафиксированные в файле `requirements.txt`:
- `streamlit` (UI-каркас)
- `pandas` (Работа с таблицами и фильтрацией)
- `pulp` (Математическое MILP-ядро для инициализации оптимизатора CBC)
- `altair` (Интерактивная статистика и графики)
- `openpyxl` & `xlsxwriter` (Движки для генерации и чтения Excel файлов)

Установка всех зависимостей:
```bash
pip install -r requirements.txt
```

## Ограничения версии 1.0 (Known Limitations)
1. **Rule-Based Explainer**: Текстовые объяснения (на вкладке *Decision Explanations*) базируются на жестких эвристиках и метриках оптимального решения. Если установка была отложена из-за конкуренции с другими установками на фоне свободного трубопровода, система классифицирует это как "Inferred / Likely tie-breaker", так как не парсит дерево решений солвера напрямую.
2. **Типизация ввода**: При загрузке Excel или CSV заголовки колонок должны строго совпадать с английскими наименованиями (`co2_flow_mtpy`, `capture_cost_euro_per_t` и т.д.).
3. **Эквивалентный оптимум (Tie-Breaker):** В текущей настройке отсутствует штраф за поздний старт. Если установке достаточно 10 лет для отработки в течение 25-летнего горизонта (при неограниченном хабе), солвер CBC выберет абсолютно **любой** математически подходящий год запуска. Штраф искусственно закомментирован в математическом ядре для сохранения академической чистоты целевой функции.
